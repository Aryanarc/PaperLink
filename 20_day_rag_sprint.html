20-day sprint
ArXiv Research Assistant —
Chat with 100+ Papers, Live
A focused RAG system specifically for research papers. Upload PDFs or paste arXiv links → ask questions across your entire library → get cited answers. Clean chat UI, source panel, RAGAS score shown live. Fully vibe-coded frontend.
Why this version, not the full one
Scoped to win
No multimodal OCR complexity. Text-only PDFs keep the pipeline clean and shippable in 20 days without shortcuts.
Still impressive
Hybrid search + reranker + RAGAS eval is still rare at student level. The demo is identical — nobody notices what you cut.
Vibe-code friendly
Backend is ~300 lines of Python. Frontend is a single React app. You can ship the whole thing with AI assistance cleanly.
Demo-able in interviews
ArXiv papers are universally understood. Every AIML interviewer immediately gets the use case and can test it themselves.
Day-by-day plan — click each phase
Days 1–4
Core pipeline
backend▶
Day 1
Project setup. Repo, venv, FastAPI skeleton, folder structure. Install PyMuPDF, LangChain, ChromaDB, sentence-transformers. Vibe: ask Claude to scaffold the full folder structure and FastAPI app in one shot.
Day 2
PDF ingestion. Upload PDF → extract text with PyMuPDF → chunk with RecursiveCharacterTextSplitter (chunk_size=512, overlap=64) → store chunks with metadata (page, source filename).
Day 3
Embeddings + vector store. Embed chunks with BGE-small-en (free, fast, strong). Upsert into ChromaDB with metadata. Build a query endpoint that returns top-5 chunks.
Day 4
LLM answer generation. Pull Mistral-7B via Ollama (runs locally, free). Build the prompt template: system prompt + retrieved chunks + user question → stream the answer back. Test in Postman/curl.
Days 5–8
Retrieval quality
ML core▶
Day 5
Hybrid search. Add BM25 retrieval alongside dense embeddings using rank_bm25. Implement Reciprocal Rank Fusion to merge both result lists. Your recall goes up immediately.
Day 6
Cross-encoder reranker. After retrieval, rerank top-15 chunks using cross-encoder/ms-marco-MiniLM-L-6-v2 (HuggingFace). Keep top-4 for the LLM. This is what separates your project from 95% of student RAG.
Day 7
ArXiv direct import. Accept an arXiv link → fetch PDF via arxiv API → auto-ingest. User never touches a file. Vibe: Claude writes this endpoint in 10 minutes.
Day 8
RAGAS evaluation. Build a 20-question test set on 5 papers. Run faithfulness + answer_relevancy + context_precision. Log scores to a JSON file. You now have real numbers to put on your resume.
Days 9–14
Frontend — the vibe coding phase
UI/UX▶
Day 9
React app scaffold. Vite + React + Tailwind. Two-panel layout: left = document library, right = chat window. Vibe: describe the layout to Claude, get the full component tree.
Day 10
Upload + library panel. Drag-and-drop PDF upload, arXiv URL input box, list of ingested papers with page count and status badges.
Day 11
Chat UI with streaming. Message bubbles, typing indicator, stream the LLM response token-by-token using SSE (Server-Sent Events). Feels like ChatGPT.
Day 12
Source citation panel. Every answer shows which chunks were used — paper name, page number, highlighted excerpt. This is the single most impressive UI feature in the demo.
Day 13
Metrics dashboard. A small panel showing RAGAS scores (faithfulness, relevancy, precision) as gauge charts. Shows you actually evaluated it, not just built it.
Day 14
UI polish. Dark mode, loading skeletons, empty states, error handling, smooth transitions. Vibe: this entire day is just prompting Claude with screenshots and saying "make this feel premium."
Days 15–17
Deploy + document
ship it▶
Day 15
Dockerize. docker-compose with FastAPI backend + ChromaDB volume + React build. Anyone can run it with one command. Vibe: Claude writes the entire Dockerfile and compose file.
Day 16
Deploy. Backend on Render free tier (or HuggingFace Spaces). Frontend on Vercel. Add a live link to your README and LinkedIn. Interviewers can use it before you walk in.
Day 17
README + architecture diagram. Clean README with a system diagram, demo GIF, RAGAS scores, and setup instructions. This is what they see first on GitHub.
Days 18–20
Interview prep
placement▶
Day 18
Record the demo video. 90-second screen recording. Upload paper → ask 3 questions → show citations → show RAGAS panel. Post on LinkedIn. This alone can get recruiter DMs.
Day 19
Write your 2-minute explanation. Practice saying: what it does → why retrieval is hard → how you solved it (hybrid search + reranker) → how you measured it (RAGAS). No jargon, just clarity.
Day 20
Deep-dive prep. Be ready for: "Why chunk size 512?", "Why BM25 + dense?", "What did RAGAS tell you?", "What would you improve?". Knowing your own choices is what separates builders from copiers.
